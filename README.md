# 星芽岛 iPad 网页版 v1.2.6 正式部署版

本版本由已经测试通过的 v1.2.5 整理而成，适用于 GitHub Pages 和 iPad 主屏幕网页应用。

## 保留原数据

正式版继续使用原来的浏览器数据键：

starSprout-iPad-production-v1

只要继续使用原来的 GitHub Pages 域名和网址，原来的以下数据会保留：

- 前四座岛建设进度
- 家长添加的任务
- 任务目标次数
- 固定任务与临时任务设置
- 当前任务完成次数
- 建设记录
- 当前所在岛屿

首次运行新版本时，程序会在旧数据上补充第五座雨林岛的数据，不会覆盖原数据。

## 第五座雨林岛

雨林岛在椰子岛达到 60 / 60 后正常解锁。

雨林岛界面与前四座岛一致：

- 左侧：今日任务
- 中央：自由建造雨林岛
- 底部：岛屿地图、建设进度、家长设置

每完成一次任意任务，会从尚未解锁的雨林素材中随机解锁一种。
初始开放 12 种基础素材，共有 80 种素材。

## GitHub Pages 上传

请把本压缩包解压后的内容上传到原仓库发布目录。

仓库根目录应能直接看到：

- index.html
- styles.css
- app.js
- rainforest.js
- manifest.webmanifest
- service-worker.js
- assets 文件夹

不要把外层文件夹或 ZIP 文件本身当作网站内容上传。

上传完成后点击 Commit changes，等待 GitHub Pages 重新部署。

## iPad 更新

更新后请继续使用原来的 GitHub Pages 地址。

如果仍看到旧版：

1. 在 Safari 打开原网站并刷新两次
2. 等待一两分钟让新 Service Worker 接管
3. 主屏幕图标仍显示旧内容时，关闭应用后重新打开
4. 必要时删除主屏幕图标，再从同一网址重新添加到主屏幕

删除主屏幕图标通常不会自动删除 Safari 网站数据，但不要手动清除 Safari 的网站数据。
