# 星芽岛 iPad 网页版 v1.1.1（任务可重复完成）

本版本在不改变原主页面布局的前提下完成：

- 今日任务数量不限
- 每个任务目标次数支持 1–99
- 每点击一次“完成一次”，当前岛屿建设进度增加 1 点
- 每座岛仍保持 60 / 60 上限
- 新增第四座岛“椰子岛”
- 海底邮局岛完成后解锁椰子岛
- 椰子岛按阶段建设：
  1. 蔚蓝海面
  2. 建设小岛与沙滩
  3. 海星、螃蟹、海龟和贝壳出现
  4. 种下椰子树
  5. 长出树叶
  6. 长出椰子
  7. 长出更多椰子树
  8. 60 / 60 显示完整椰子岛
- 保留原有浏览器本地进度，不更换 localStorage 数据键
- 更新 PWA 离线缓存，适合 GitHub Pages 和 iPad 主屏幕使用

## 上传到 GitHub Pages

请把本压缩包解压后的所有文件上传到 GitHub 仓库根目录，确保以下文件与 assets 文件夹位于同一级：

- index.html
- styles.css
- app.js
- manifest.webmanifest
- service-worker.js
- assets/

然后进入 GitHub 仓库：

1. Settings
2. Pages
3. Build and deployment
4. Source 选择 Deploy from a branch
5. Branch 选择 main，目录选择 /(root)
6. 保存并等待部署完成

## iPad 更新方式

上传新版本后，第一次打开可能仍看到旧缓存。建议：

1. 在 Safari 中打开 GitHub Pages 地址
2. 刷新页面两次
3. 若主屏幕旧图标仍显示旧版，先删除旧主屏幕图标
4. 在 Safari 分享菜单中重新选择“添加到主屏幕”

任务、岛屿进度和建设记录保存在当前网站域名的 Safari 本地存储中。只要 GitHub Pages 地址不变，通常会继续保留原进度。

## v1.1.1 新增行为

- 每个任务达到目标次数后仍可继续点击。
- 达标后的按钮显示“再完成一次”。
- 例如“英语听读”目标为 1 次，当天仍可完成第 2、3、4 次。
- 每一次重复完成都会让当前岛屿建设进度增加 1 点，岛屿仍以 60 / 60 为上限。
- 任务卡可以显示超过目标的次数，例如 4 / 1。
- “今日星光”会统计所有实际完成次数，因此可能显示为 7 / 4。
- 第二天固定任务的完成次数仍会自动归零。
- 未修改原有网页布局，也未更换本地存储键。
